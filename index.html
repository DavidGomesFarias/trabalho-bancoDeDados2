Indexação de banco de dados:

A indexação é uma técnica usada para melhorar a velocidade de recuperação de dados em um banco de dados. Assim como em um livro, onde um índice permite encontrar informações rapidamente sem precisar ler todas as páginas, um índice em um banco de dados permite acessar linhas específicas de uma tabela sem precisar varrer a tabela inteira. Isso é particularmente útil em tabelas grandes.

Conceito de Índices:

Índices são estruturas de dados criadas sobre uma ou mais colunas de uma tabela, destinadas a otimizar as operações de busca, como consultas (SELECT), filtragem (WHERE), junções (JOIN), e ordenações (ORDER BY). Eles armazenam uma cópia ordenada dos dados selecionados das colunas indexadas, permitindo acesso mais rápido.

B-tree Index (Índice em Árvore B):

O índice B-tree é o tipo de índice mais comum nos sistemas de gerenciamento de banco de dados (SGBDs). Ele organiza os dados em uma estrutura de árvore balanceada, onde os nós representam blocos de dados. Essa árvore permite que o SGBD encontre os registros de forma eficiente, mesmo em tabelas grandes. A busca, inserção, e exclusão de registros têm complexidade logarítmica, o que proporciona boa performance para uma ampla variedade de consultas.

Hash Index:

O índice hash é usado para busca de igualdade, ou seja, quando a consulta é do tipo "igual a" (ex.: `WHERE campo = valor`). Ele usa uma função de dispersão (hash) para mapear valores de índice diretamente para um local no disco, permitindo buscas muito rápidas. No entanto, ele não é eficiente para consultas que envolvem intervalos (ex.: `WHERE campo > valor`), pois os valores são armazenados de forma não ordenada.

Bitmap Index:

O índice bitmap é utilizado principalmente em tabelas com colunas de poucos valores distintos (alta cardinalidade). Ele armazena uma série de bits, onde cada bit indica a presença ou ausência de um determinado valor em uma linha. Esses índices são particularmente eficientes em consultas complexas que envolvem múltiplas condições (ex.: `WHERE` com várias colunas), já que podem combinar bitmaps rapidamente usando operações lógicas (AND, OR). Eles são mais comuns em ambientes de data warehouse.
